1、主从模式
从节点 出发sync 生成rdb 快照同步到从节点
期间产生的写操作，会写入到缓存汇总，等待rdb 写入晚，顺序写缓存数据

2、哨兵模式
依赖于主从模式，部署多个监控节点，当主节点挂了 经过选举选择出新的master 节点

3、集群模式
将不同的数据推送到不同的节点去，实现redis 的横向扩展, 坏处是如果某一个机器的主从都挂了，整个机器都挂了，
